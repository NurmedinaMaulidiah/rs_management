Sistem manajemen pasien untuk Rumah Sakit:

Admin: CRUD user, CRUD pasien, CRUD layanan, CRUD layanan yg diberikan dokter

Staff: CRUD pasien, lihat daftar pasien, detail pasien (tanpa rekam medis).

Dokter: Lihat pasien yang ditangani, tambah/ubah rekam medis pasien.

Hapus Pasien

- RELASI DAN PENGHAPUSAN DATA (DATABASE RUMAH SAKIT)

Kesimpulan Proses Hapus Data di Sistem RS

1. Hapus Dokter (User role = dokter)
Sebelumnya: Hapus dokter → pasien dan rekam medis ikut terhapus → masalah.
Sekarang:
Dicek dulu apakah dokter masih memiliki pasien.
Jika masih ada pasien → tidak bisa hapus dokter.
Pasien tetap ada, rekam medis tetap ada.
✅ Aman, data pasien tidak hilang sembarangan.

2. Hapus Layanan RS
Sebelumnya: Hapus layanan → error atau pasien ikut hilang → masalah.
Sekarang:
Dicek dulu apakah layanan masih digunakan oleh pasien.
Jika masih dipakai pasien → layanan tidak bisa dihapus.
✅ Aman, integritas data pasien tetap terjaga.

3. Hapus Layanan yang Diberikan Dokter (doctor_services)
Menghapus relasi dokter → layanan tidak menghapus pasien.
Nama dokter dan nama layanan di tabel pasien tetap ada.
Jika perlu, pasien dapat diedit untuk mengganti dokter/layanan.
✅ Aman, normal, dan tidak menimbulkan masalah.

4. Hapus Rekam Medis
Hapus rekam medis → hanya rekam medis yang terhapus, pasien tetap ada.
✅ Aman, sesuai logika CASCADE.

5. Hapus Pasien
Hapus pasien → seluruh data pasien termasuk rekam medis ikut terhapus (ON DELETE CASCADE).
✅ Aman, logis, data rekam medis tergantung pasien.

6. Edit layanan dokter
- Jika layanan dokter diubah (misalnya dari 2 layanan menjadi 1 layanan):
- Perubahan hanya berlaku untuk data relasi dokter-layanan ke depan.
- Data pasien lama tidak ikut berubah.
- Pasien yang sebelumnya sudah terdaftar:
- Tetap memiliki dokter dan layanan lama di tabel patients.
- Masih bisa dilihat oleh dokter tersebut.
- Dokter masih bisa menambahkan rekam medis pasien tersebut.
- Jika ingin menyesuaikan:
Admin/Staff dapat mengedit data pasien (mengganti dokter atau layanan).
Setelah diganti, dokter baru dapat melihat dan melanjutkan rekam medis pasien tersebut.


KESIMPULAN:

* CASCADE digunakan untuk data yang benar-benar bergantung (contoh: rekam medis pada pasien).
* SET NULL digunakan agar data utama (pasien) tetap aman walaupun relasi dihapus.
* Tujuan utama adalah menjaga integritas data agar tidak terjadi data yang tidak valid atau hilang secara tidak sengaja.
